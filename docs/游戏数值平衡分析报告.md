## 二、九张地图路径覆盖分析

### 2.1 判定标准

- **射程换算**：英雄射程 N 格 = `N × (cellSize + gap) + cellSize × 0.25` 像素
- **优秀目标**：所有非路径格到路径的最小曼哈顿距离 ≤ 4
- **曼哈顿距离 = 5**：仅二郎神等超远射程单位覆盖，不再接受为常规可用格
- **曼哈顿距离 ≥ 6**：任何英雄都打不到，属于死区

### 2.2 总览

| 地图 | 尺寸 | 原路径长 | 新路径长 | 最大距离 | 距离 5 格 | 评级 |
|------|------|----------|----------|----------|-----------|------|
| Ch1 山门 | 6×8 | 11 | 13 | 4 | 0 | 🟢 优秀 |
| Ch2 流沙河 | 6×8 | 11 | 15 | 4 | 0 | 🟢 优秀 |
| Ch3 密林 | 7×7 | 14 | 14 | 3 | 0 | 🟢 完美 |
| Ch4 悬崖 | 7×8 | 13 | 15 | 4 | 0 | 🟢 优秀 |
| Ch5 石阵 | 8×8 | 17 | 19 | 4 | 0 | 🟢 优秀 |
| Ch6 沼泽 | 7×9 | 12 | 18 | 4 | 0 | 🟢 优秀 |
| Ch7 沙漠 | 8×9 | 15 | 17 | 4 | 0 | 🟢 优秀 |
| Ch8 云霄 | 8×8 | 16 | 18 | 4 | 0 | 🟢 优秀 |
| Ch9 终局 | 8×9 | 17 | 17 | 4 | 0 | 🟢 优秀 |

> 注：所有路径保留原起点和终点，路径连续且不重复经过同一格。初始开放格仍由算法选择靠近前 4 个路径点的位置。

### 2.3 最新修正策略

路径从“偏一侧推进”改为更明显的 S/Z 形覆盖。宽图优先让路径在水平方向多摆动，避免左上、右上、右下角落只有二郎神能覆盖。

| 地图 | 主要改动 | 收益 |
|------|----------|------|
| Ch1 | 顶部延伸到 col 3，底部回收到 col 2 | 右上角和左下角都降到 ≤4 |
| Ch2 | 顶部向左覆盖，随后向右下摆到 col 6 | 同时覆盖左上和右下 |
| Ch4 | 顶部左移到 col 4，底部左移到 col 2 | 左侧角落降到 ≤4 |
| Ch5 | 顶部延伸到 col 3，保留中段回折 | 右上角降到 ≤4 |
| Ch6 | 顶部覆盖 col 4，中央回折到 col 3，右下经过 col 6 | 最严重的 3 个死区全部消除 |
| Ch7 | 顶部延伸到 col 3-6，右侧纵向下行 | 右上和右下都降到 ≤4 |
| Ch8 | 顶部延伸到 col 4 后再回到右侧 | 左上死区消除 |
| Ch9 | 顶部延伸到 col 4 | 右上边缘格降到 ≤4 |

路径变长会让部分章节变简单，尤其 Ch6 从 12 格变为 18 格，敌人行进时间明显增加。这个收益是有意的，因为原 Ch6 同时存在最严重死区和最短路径，难度波动过大。

如果后续实测这些章节偏简单，优先微调该章节敌人血量、速度或波次密度，不建议把路径退回距离 5。地图覆盖是站位公平性的底层约束，应该比单关难度更稳定。

### 2.4 路径设计原则

从本轮路径修正总结：

1. 所有非路径格到路径的最小曼哈顿距离必须 ≤ 4。
2. 路径应覆盖棋盘主要区域，避免大面积空白。
3. S 形或 Z 形蜿蜒优于直线斜穿。
4. 宽图需要路径在水平方向有足够摆动，不能偏居一侧。
5. 路径不能重复经过同一格，避免怪物绕圈和箭头表现混乱。
6. 起点和终点尽量保持不变，避免破坏章节主题和唐僧终点位置。

## 三、核心问题诊断

### 🟡 问题 1：小兵 Rank 需要结合合成成本评估

R1 弓手 ATK 20，R5 弓手 ATK 210，单体攻击差距为 **10.5 倍**。但 R5 理论上需要 16 个同类 R1 合成，不能只用单体倍率判断强弱。

当前更准确的问题是：高阶兵的**格子效率**可能过强，而不是低阶兵完全无效。是否要下调 `1.8`，应结合召唤成本、占格压力、存活时间和每波 TTK 一起判断。

在英雄/Boss 技能和友方受击逻辑补齐前，不建议把小兵倍率下调作为 P0。否则可能先削弱合成爽感，后续再加受击和 Boss 技能时难度会二次上升。

### 🟢 问题 2：英雄碎片获取过慢（已缓解）

旧版选 3 个英雄时，特定英雄碎片单卡概率仅 **2.7%**，激活 1 个核心英雄约需要 **22 次召唤**。现已改为英雄碎片 22%、选中英雄权重 5×，并保留前 3 次付费召唤孙悟空碎片保底。

后续观察点是：定向养成不能太快把 14 名英雄全部拉满，但首个核心英雄必须稳定启动。

### 🟢 问题 3：开山斧爆率太低（已用保底处理）

- 单卡 12%，5 连抽期望 0.6 把
- 5 连抽至少 1 把约 47%，且有早期付费召唤保底
- 每关通常有 5-6 个锁定格子，Journey 有 81 关
- 玩家可能面临"有桃子但没斧头开格子"的尴尬

现已加入第 4 次付费召唤早期保底，并加入 12 张非斧卡后的长期轻保底。抽卡池只保留开山斧这一种道具，其他战术道具迁移到法宝系统，避免局内抽卡目标混杂。

### 🟢 问题 4：击杀奖励无差异化（已修正）

旧版所有怪物击杀都只给 5 🍑，击杀一只熔岩怪和击杀大鹏金翅雕回报一样。现已改为：

- 普通小妖：3 🍑
- 精英：12 🍑
- Boss：45 🍑

这样普通怪海不会过度放大仙桃收入，精英和 Boss 节点也更有击杀反馈。

### 🟢 问题 5：Journey 模式后期数值（已改为质量替代数量）

旧版 Journey Level 81 第 1 波有 **31 个敌人**，spawn interval 0.78s，DPS 压力和性能风险都偏高。

现已改为按章节倍率提升敌人 HP/攻击，数量基础上限为 12。Level 81 第 1 波从 31 个降到 12 个，但敌人 HP 会吃到第 9 章 3.6× 倍率，后期更像“少量高质量压力”。

### 🟡 问题 6：地图覆盖已修正，需复核路径变长后的难度

九张地图已经调整到所有非路径格最小曼哈顿距离 ≤ 4，距离 5 的边缘格也已清零。详见[第二章](#二九张地图路径覆盖分析)。

新的关注点是路径变长后的难度变化。Ch6 从 12 格变为 18 格，敌人行进时间明显增加；如果实测偏简单，应通过怪物速度、血量或波次密度补偿，而不是回退路径覆盖。

### 🟢 问题 7：英雄和 Boss 缺少技能，战斗同质化（已加入原型）

已加入第一版自动技能原型：孙悟空 Boss 爆发、观音低血治疗和护盾、红孩儿范围清线和灼烧、二郎神高威胁狙击、哪吒周期多目标。部分已有被动也已补实，例如二郎神优先高威胁目标、牛魔王低血量增伤和减伤、太上老君灼烧、蜘蛛精减速光环、托塔天王定身。

Boss 已补齐 7 类机制：黑熊精护盾/回血/反伤、金角大王吸收单体和伤害光环、红孩儿灼烧光环和火球、白骨夫人召唤小怪和变形不可选中、青狮咆哮眩晕和冲锋、白象范围践踏和伤害抵抗、大鹏金翅雕飞行加速和后排风刃。

精英能力也已承接到战斗逻辑：黄风怪眩晕攻击、狐狸精闪避、象兵践踏、大鹏鹰速度光环。Journey 章节 Boss 覆盖 7 个 Boss，Defense 无尽模式会轮换 4 种精英和 7 个 Boss。

### 🟢 问题 8：友方血条缺少受击闭环（已补齐）

小兵和英雄都有血量、血条、防御、治疗等字段。现已让敌人会攻击路径附近的友方单位：普通怪边走边打，精英和 Boss 会停下来持续攻击，八戒仍可嘲讽附近敌人。

这会让防御、治疗、坦克、回血和玉净瓶进入实际决策。后续需要通过实测确认前排死亡频率和治疗压力，避免受击逻辑把难度突然抬得过高。

## 四、优化方案

### 4.1 小兵 Rank 倍率调整：暂缓直接大砍

不建议现在把 `dmg: 1.8^(rank-1)` 直接降到 `1.5^(rank-1)`。R5 单体倍率虽然高，但它消耗 16 个同类 R1，核心收益是节省格子和操作，而不是凭空获得 10.5 倍资源。

建议先保留当前小兵成长，补齐友方受击和 Boss 技能后再复查。如果 R4/R5 仍然明显压制关卡，再做温和调整。

可选的二次调整档位：

```typescript
// 轻度调整：优先保留合成爽感
// dmg: 1.8 -> 1.7
// hp: 1.6 -> 1.55
// spd: 0.92 -> 0.94

// 中度调整：仅在高阶兵明显碾压时使用
// dmg: 1.8 -> 1.6
// hp: 1.6 -> 1.5
// spd: 0.92 -> 0.95
```

评估指标应改为：每格 DPS、16 张 R1 合成 1 个 R5 的资源回报、R5 在 Boss 技能下的存活时间，以及高阶兵对普通波和 Boss 波的 TTK。

### 4.2 爆率调整

调整前后对比：

|------|--------|--------|------|
| 英雄碎片率 | 18% | 22% | +22% |

结论：英雄碎片小幅提高，定向权重提高；开山斧作为唯一抽卡道具提供棋盘扩展保障。其他战术道具不再挤占抽卡池，转由法宝系统提供明确的 CD 和携带策略。

### 4.3 仙桃经济调整

先修正口径：当前 `SettlementSystem` 只计算结算奖励，`ResultView` 只展示奖励，并没有把奖励加入局内仙桃或存档货币。因此“通关结算是 Journey 主要收入”这个结论暂时不成立。

建议先决定结算奖励定位：

- 如果仙桃只是局内资源：结算界面不要写“获得奖励”，只展示击杀、波次、星级。
- 如果仙桃也是局外资源：需要在 `SaveData` 增加局外仙桃字段，并在结算时真正入账。

```typescript
// EnemyRewardData.ts — 区分怪物类型的击杀奖励
// 旧版: KILL_PEACH_REWARD = 5（所有类型）
// 建议: 根据 EnemyType 区分
function getKillPeachReward(enemyType: EnemyType): number {
  switch (enemyType) {
    case EnemyType.NORMAL: return 3;
    case EnemyType.ELITE:  return 12;
    case EnemyType.BOSS:   return 45;
  }
}
```

召唤消耗上限暂不建议从 160 降到 120。补齐友方受击和 Boss 技能后，玩家会把仙桃分给召唤、升级、扩容和补救道具，后期资源压力会自然上升。

> 初始仙桃保持 **50 🍑** 不变。击杀奖励从“按数量”改为“按威胁”后，普通怪海不会过度放大仙桃收入，精英和 Boss 击杀也更有反馈。

### 4.4 保底机制

private _paidSummonCount: number = 0;

早期保底解决“有桃子但没格子”的挫败，长期轻保底防极端运气。其他功能道具已经迁移到法宝系统，抽卡池保持“小兵、英雄碎片、开山斧”三类核心材料。

### 4.7 战斗机制补充：让血条和技能真正参与平衡

当前数值平衡不要只围绕“攻击力 vs 血量”。应先补齐三条机制：敌人会攻击友方单位，英雄有明确技能定位，Boss 有可读的机制压力。

#### 4.7.1 友方受击规则

当前实现已把敌人分成三类攻击行为：

| 类型 | 行为 | 目的 |
|------|------|------|
| 普通怪 | 经过路径附近时，攻击最近英雄或小兵 1 次，然后继续移动 | 让血条有意义，但不拖慢节奏 |
| 精英怪 | 会短暂停下攻击威胁最高的友方单位 | 压迫前排和高输出点 |
| Boss | 拥有周期技能，优先影响英雄、密集小兵或关键格子 | 制造布阵调整和道具使用时机 |

实现上 `BattleSystem._assignEnemyAttackTargets()` 已从“只找 Hero”改成“找所有 `Unit`”。普通怪攻击后继续走，精英和 Boss 才允许停下来持续攻击。

这样小兵和英雄的 HP、防御、回血、治疗、玉净瓶就会进入核心决策。玩家会考虑“谁放前排、谁吃技能、谁需要保护”，而不是只堆最高 DPS。

#### 4.7.2 英雄技能定位

英雄保留自动普攻，并增加低频自动技能或被动强化。第一版不做手动施法，避免移动端操作复杂。

| 英雄 | 定位 | 技能草案 |
|------|------|---------|
| 孙悟空 | Boss 破防/爆发 | 每 10 秒对 Boss 造成高额单体伤害，并附加 3 秒破甲 |
| 观音 | 治疗/保护 | 每 8 秒治疗血量最低友方，并给 1 个护盾 |
| 牛魔王 | 前排反击 | 低血量时获得减伤和反伤，吸引附近敌人 |
| 红孩儿 | 范围灼烧 | 对目标格周围释放火圈，持续伤害普通怪 |
| 二郎神 | 狙击精英 | 优先攻击精英/Boss，技能对最高血量目标追加伤害 |
| 哪吒 | 清线 | 周期性攻击 3-5 个目标，但单体倍率低于悟空 |
| 太上老君 | 持续伤害/炼丹 | 给敌人挂灼烧，击杀灼烧目标有小概率返还仙桃或经验 |

普通英雄可以做功能补位：八戒抗线、沙僧减速、白龙马穿透、黑熊反伤、蜘蛛减速、托塔控制、白骨召唤分身。

#### 4.7.3 Boss 技能定位

Boss 技能要低频、强提示、可反制。当前已按“每个 Boss 1-2 个核心机制”的原则补齐首版，避免实现复杂度和数值变量同时爆炸。

| Boss | 技能草案 | 反制方式 |
|------|---------|---------|
| 黑熊精 | 每 8 秒获得护盾并反弹部分伤害 | 等护盾结束，或用控制道具拖时间 |
| 金角大王 | 吸收最近小兵 4 秒，使其无法攻击 | 分散布阵，避免核心输出贴路 |
| 红孩儿 | 点燃 2-3 个格子，持续伤害友方 | 需要治疗、护盾或换位 |
| 白骨夫人 | 血量 50% 时召唤小怪并短暂无敌 | 需要清线能力 |
| 青狮 | 咆哮眩晕附近友方 2 秒 | 远程分散站位 |
| 白象 | 践踏路径周围格子，造成范围伤害，并常驻伤害抵抗 | 前排需要血量和治疗 |
| 大鹏金翅雕 | 短暂加速并攻击后排远程 | 需要控场和后排保护 |

Boss 血量应在技能加入后再最终确定。没有技能时建议先提高 2-3 倍；有技能后可降低纯血量，依靠技能制造压力。

#### 4.7.4 技能加入后的再平衡顺序

1. 已实现敌人攻击友方单位，血条、治疗、防御开始影响战斗。
2. 已补齐 7 个 Boss 技能，用黑熊、金角、红孩儿、白骨夫人、青狮、白象、大鹏覆盖护盾、范围伤害、控制/吸收、召唤、位移和后排压力。
3. 已补核心英雄技能，覆盖单体爆发、清线、治疗/护盾、控制和高威胁狙击。
4. 已补精英能力和 Boss/精英轮换，并用测试锁住；后续实测再重算怪物血量、英雄 DPS、小兵成长和仙桃收入。

这能避免“先调数值，后加技能，再全部推倒重来”。技能会显著改变难度曲线，所以最终数值应放在机制闭环之后。

## 附录：伤害计算链路

```
英雄最终伤害 = baseAttack
  × levelMultiplier(1 + (level-1) × 0.15)
  × (1 + factionDamageBonus)
  × heroSpecificMultiplier (如孙悟空 1.5 vs boss)
  × critMultiplier (1.5 if crit)
  × damageVariance(±10%)

英雄实际 DPS ≈ 英雄最终伤害
  × attackSpeed
  × (1 + (level-1) × 0.03)
  × (1 + factionAttackSpeedBonus)

小兵最终伤害 = baseAttack
  × rankMultiplier(1.8^(rank-1))
  × damageVariance(±10%)

敌方对单位伤害 = enemy.attack - unit.defense (min 1)

敌方对唐僧伤害 = 1 per enemy that reaches end
```

## 修订结论

> 📌 **核心结论**：当前不应先做大幅数值削弱。项目更底层的问题是 **战斗机制闭环不完整**。
>
> 1. 英雄和 Boss 缺少技能，导致差异主要体现在 DPS、射程、血量上。
> 2. 友方血条没有稳定受击闭环，治疗、防御、坦克、回血和道具价值没有充分释放。
> 3. 结算奖励目前只是展示，没有真正入账，仙桃经济需要先统一口径。
> 4. 小兵 Rank 倍率需要结合 16 合 1 的成本看，不适合直接作为 P0 大砍。
>
> 本轮已完成 P0/P1 和主要 P2 项：结算口径、友方受击、地图覆盖、Boss/英雄技能原型、精英/Boss 血量、召唤爆率、开山斧保底、Journey 后期质量化都已落地。下一轮重点应放在实机验证：观察前排死亡频率、Boss TTK、前 10 波召唤次数、Journey 60-81 关性能和三星率，再决定是否温和下调小兵 Rank 倍率。
