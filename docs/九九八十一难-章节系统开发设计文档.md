# 九九八十一难 — 章节系统开发设计文档

> 版本：v1.0 | 日期：2026-07-07 | 状态：待开发

## 一、设计目标

将当前「八十一难」入口进去后 81 关蛇形平铺的单一视图，重构为 **"九章 → 关卡" 两级导航结构**：

- **第一级（章节选择）**：展示 9 个章节卡片，每章含名称、解锁状态、进度条、通关奖励预览
- **第二级（关卡网格）**：选中章节后展开，展示该章的 9 个关卡节点（Boss 关高亮 + 通关奖励标注）
- **章末 Boss 关（第 9 关）**：通关解锁对应法宝 + 英雄，作为章节核心目标驱动

### 2.3 章节进度计算

章节状态分三种：

| 状态 | 条件 | 视觉表现 |
|------|------|----------|
| **锁定** | 前一章未通关 | 灰色卡片，显示 🔒 |
| **进行中** | 已解锁但 Boss 关未通 | 主题色卡片，显示进度条 |
| **已完成** | Boss 关已通 (clearedLevels 包含 bossLevelId) | 金色边框 + ✓ 标记，可扫荡 |

```typescript
function getChapterState(chapterId: number, levelData: LevelData): 'locked' | 'active' | 'completed' {
  if (chapterId === 1) {
    // 第一章始终解锁
    if (levelData.isCleared(9)) return 'completed';
    return 'active';
  }
  const prevChapterBossLevel = (chapterId - 1) * 9; // 前一章 Boss 关编号
  if (!levelData.isCleared(prevChapterBossLevel)) return 'locked';
  if (levelData.isCleared(chapterId * 9)) return 'completed';
  return 'active';
}
```

## 四、界面布局设计

### 4.1 章节选择界面（ChapterSelectView）

**核心设计原则：奖励前置展示。** 每张章节卡片必须直接展示 Boss 关通关后可获得的英雄和法宝，让玩家一眼就知道打通的收益，形成明确的推进动力。

**设计要点：**
- 手机竖屏 750×1334
- **9 张卡片无法同时完整显示**，使用 Phaser 遮罩 + 触控拖拽实现纵向滚动：
  - 遮罩区域：x=34, y=220, w=682, h=792（复用现有 `_drawBackground` 内容区）
  - 卡片总高度：9 × (126+16) - 16 = 1262px（超出遮罩区 470px）
  - 触控拖拽：监听 `pointermove`，`scrollY` 随拖拽偏移，clamp 到 [0, 1262-792]
  - 惯性：拖拽释放后 `scene.tweens` 带 deceleration 惯性滚动
  - 滚动指示器（可选）：右侧细条显示当前位置
- 每卡高度 **120px**（含内边距 18px），卡片间距 **16px**
  - 标题区 36px + 进度条 14px（含 4px 上下间距）+ 奖励区 52px = 合计 OK
- **奖励展示区是卡片核心视觉焦点**
- 锁定章节不展示具体奖励，仅显示解锁条件文字居中占位

### 4.3 奖励图标渲染方案

三种尺寸规格，同一套渲染逻辑，调用 `VisualPainter` 已有能力绘制：

| 场景 | 英雄头像 | 法宝图标 | 容器形状 |
|------|---------|---------|---------|
| 章节卡片奖励区 | 直径 72px | 72×72px | 圆形 / 圆角方形 |
| Boss 关节点 mini 条 | 直径 18px | 18×18px | 圆形 / 圆角方形 |
| 关卡详情奖励区 | 直径 96px | 96×96px | 圆形 / 圆角方形 |

**实现思路：**

```typescript
// 在 ChapterSelectView / LevelGridView 中绘制奖励图标

// 英雄头像：圆形 + 首字（用 Graphics 画圆 + Text 写首字）
function drawHeroIcon(scene, x, y, size, heroConfig, owned) {
  const g = scene.add.graphics();
  const r = size / 2;
  g.fillStyle(owned ? heroConfig.fill : 0x49505f, 1);
  g.fillCircle(x, y, r);
  g.lineStyle(2, owned ? 0xf0c15a : 0x6f7785, 0.9);
  g.strokeCircle(x, y, r);
  // 英雄首字居中
  const char = scene.add.text(x, y, heroConfig.name[0], {
    fontSize: `${size * 0.4}px`, color: '#fff', fontStyle: 'bold'
  });
  char.setOrigin(0.5);
  // 未获得：半透明灰度遮罩 + 🔒 角标
  if (!owned) {
    g.fillStyle(0x000000, 0.45);
    g.fillCircle(x, y, r);
  }
}

// 法宝图标：圆角方形 + 首字
function drawArtifactIcon(scene, x, y, size, artifactConfig, owned) {
  const g = scene.add.graphics();
  g.fillStyle(owned ? 0x1d2f43 : 0x29303a, 1);
  g.fillRoundedRect(x - size/2, y - size/2, size, size, 10);
  g.lineStyle(2, owned ? 0xf0c15a : 0x6f7785, 0.9);
  g.strokeRoundedRect(x - size/2, y - size/2, size, size, 10);
  const char = scene.add.text(x, y, artifactConfig.name[0], {
    fontSize: `${size * 0.4}px`, color: '#ffd36a', fontStyle: 'bold'
  });
  char.setOrigin(0.5);
  if (!owned) {
    g.fillStyle(0x000000, 0.45);
    g.fillRoundedRect(x - size/2, y - size/2, size, size, 10);
  }
}
```

> 后续替换正式美术资源时，只需将 `Graphics.fillCircle` / `fillRoundedRect` 替换为 `scene.add.image(x, y, heroIconKey)`，接口不变。

## 四-PRO、UI/UX Pro Max 优化设计规范

> 本节省略了原始设计方案中"六、视觉设计规范"和"七、动效设计"的通用描述，
> 替换为经过 UI/UX Pro Max 规则系统校验的精确像素级规范。每条标注了对应的规则来源。

### 4P.1 设计风格定位

| 维度 | 选择 | 依据 |
|------|------|------|
| **风格** | Flat Design Mobile (Touch-First) | 规则 §4 `style-match`：2D 棋盘塔防手游，无阴影、平涂色块、几何装饰 |
| **色调** | Dark-only（暗色底 + 金/青绿/朱红强调）| 国风西游 + 棋盘 UI 传统，无 light mode 需求 |
| **密度** | Standard (16-64px 间距体系) | 章节卡片 + 关卡网格，信息密度适中 |
| **动效等级** | Standard (stagger + press feedback + transitions) | 规则 §7：150-300ms 微交互，不可装饰性动画 |

**反模式警告（规则 §4）：**
- ❌ 不要在平涂暗色 UI 中使用模糊/毛玻璃效果 → 与整体 flat 风格冲突
- ❌ 不要混合多种风格：章节选择和关卡网格必须统一 flat 暗色体系
- ❌ 不要用 Emoji 做图标：锁 🔒、完成 ✓ 需要用 Graphics 绘制矢量图标，而非 Emoji 字符

### 4P.2 精确间距体系

遵循规则 §5 `spacing-scale` 的 8px 递增系统：

```
--space-xs:   8px   (关卡节点间最小间距)
--space-sm:  12px   (卡片内边距、图标间隙)
--space-md:  16px   (卡片间距、区块间距)
--space-lg:  24px   (主区块上下间距)
--space-xl:  32px   (标题与内容区间距)
--space-2xl: 48px   (页面顶部/底部留白)
```

**应用到章节选择页：**

| 元素 | 间距值 | 空间标记 |
|------|--------|----------|
| 顶部标题栏 → 第一张卡片 | 32px | `--space-xl` |
| 章节卡片之间 | 16px | `--space-md` |
| 卡片内部：标题区 → 进度条 | 12px | `--space-sm` |
| 进度条 → 奖励展示区 | 16px | `--space-md` |
| 奖励区内：英雄图标 ↔ 法宝图标 | 16px | `--space-md` |
| 卡片内边距（四周） | 20px | 固定内边距 |

**应用到关卡网格页：**

| 元素 | 间距值 | 空间标记 |
|------|--------|----------|
| 关卡节点间水平间距 | 72px（保持现有蛇形布局） | 不变 |
| 关卡节点间垂直间距 | 76px | 不变 |

#### 4P.9.5 奖励解锁动画（章节通关后首次展示）

```typescript
// 通 Boss 关后返回章节选择时触发
// 规则 §7: motion-meaning — 因果明确（通关 → 解锁英雄+法宝）
// 只播放一次（用 LevelData 做标记）

function playUnlockReveal(card: ChapterCard): void {
  // 阶段 1: 卡片金色光晕爆发 (300ms)
  this.scene.tweens.add({
    targets: card.glowGraphics,
    alpha: { from: 0, to: 0.8 },
    scale: { from: 0.5, to: 1.5 },
    duration: 300,
    ease: 'Cubic.Out',
    onComplete: () => {
      // 阶段 2: 英雄+法宝图标弹出 (stagger 80ms)
      this.scene.tweens.add({
        targets: card.heroIcon,
        scale: { from: 0, to: 1 },
        alpha: { from: 0, to: 1 },
        duration: 280,
        ease: 'Back.Out',
      });
      this.scene.tweens.add({
        targets: card.artifactIcon,
        scale: { from: 0, to: 1 },
        alpha: { from: 0, to: 1 },
        duration: 280,
        ease: 'Back.Out',
        delay: 80,                  // stagger between hero & artifact
      });
    },
  });
}
```
