## 一、设计目标 ✅

将「八十一难」入口进去后 81 关蛇形平铺的单一视图，重构为 **"九章 → 关卡" 两级导航结构**：

- **第一级（章节选择）**：展示 9 个章节卡片，每章含名称、解锁状态、进度条、通关奖励预览 **(已实现：ChapterSelectView)**
- **第二级（关卡网格）**：选中章节后展开，展示该章的 9 个关卡节点（Boss 关高亮 + 通关奖励标注）**(已实现：LevelGridView)**
- **章末 Boss 关（第 9 关）**：通关解锁对应法宝 + 英雄，作为章节核心目标驱动 **(已实现：LevelData.awardChapterClear)**
- **章节主题棋盘**：每章独立棋盘配色 + 锁定格障碍物 + 特殊机制 **(已实现：GridManager + ChapterMechanicSystem)**

### 2.2 ChapterConfig 完整类型定义（实际代码）

```typescript
// src/config/ChapterConfig.ts

export type ChapterState = 'locked' | 'active' | 'completed';
export type MechanicType =
  | 'none' | 'silence' | 'respawn' | 'fire_zone'
  | 'burn_all' | 'slow' | 'root' | 'stats_boost' | 'shield_stack';

// 棋盘配色（每章独立）
export interface ChapterBoardColors {
  panelStroke: number;       // 面板边框
  buildableFill: number;     // 可布阵格填充
  buildableBorder: number;   // 可布阵格边框
  pathFill: number;          // 路径格填充
  pathBorder: number;        // 路径格边框（统一朱砂 #c43d30）
  lockedFill: number;        // 锁定格填充
  lockedBorder: number;      // 锁定格边框
  pathWideLine: number;      // 路径宽线
  pathMidLine: number;       // 路径中线
}

// 章节特殊机制
export interface ChapterMechanic {
  type: MechanicType;
  interval?: number;   // 触发间隔（秒）
  duration?: number;   // 持续时间（秒）
  value?: number;      // 效果数值（百分比）
}

export interface ChapterConfig {
  chapterId: number;
  name: string;
  subtitle: string;
  levelRange: [number, number];
  bossLevelId: number;
  unlockHeroId: string;
  unlockArtifactId: ArtifactId;
  themeColor: number;
  themeColorDark: number;
  terrainType: string;
  // === 新增字段 ===
  boardColors: ChapterBoardColors;
  lockedCellImage: string;
  specialMechanic: ChapterMechanic;
}
```

九章完整数据位于 `CHAPTER_CONFIGS` 数组中，包含上述所有字段的具体值。

### 2.3 章节进度计算（实际代码）

```typescript
// src/config/ChapterConfig.ts
export function getChapterState(chapterId: number, levelData: LevelData): ChapterState {
  if (chapterId === 1) {
    return levelData.isChapterBossCleared(chapterId) ? 'completed' : 'active';
  }
  if (!levelData.isChapterBossCleared(chapterId - 1)) return 'locked';
  return levelData.isChapterBossCleared(chapterId) ? 'completed' : 'active';
}
```

### 5.3 ChapterMechanicSystem — 八种机制实现

```typescript
// src/systems/ChapterMechanicSystem.ts

export class ChapterMechanicSystem {
  constructor(mechanic: ChapterMechanic, battle: BattleSystem, grid: GridManager)

  update(dt: number): void   // 每帧由 main.ts 的 update() 调用
  destroy(): void             // 清理事件监听 + 重置全局状态

  // 八种机制实现方式：
  // 'none'        → update() 直接 return
  // 'silence'     → 定时器 → 随机选已布阵单位 → applyStun(duration)
  // 'respawn'     → 监听 ENEMY_KILLED 事件 → 概率 spawnEnemy(HP 减半)
  // 'fire_zone'   → 定时器 → 随机选 2 格 → 1s 预警 → 3s 扣血
  // 'burn_all'    → 每秒 tick → 所有已布阵单位扣 1% maxHp
  //                 → setGlobalHealMultiplier(0.5)
  // 'slow'        → 初始化时 setGlobalSpeedMultiplier(0.8)
  // 'root'        → 定时器 → 随机单位 applyStun(duration)
  // 'stats_boost' → 外部处理（main.ts 锁格+30% + transformWave 敌系数×1.15）
  // 'shield_stack'→ 定时器 → 所有场上敌人 chapterShieldStacks +1 → applyVulnerability(-10%)
}
```

### 5.5 战斗场景集成（main.ts 核心逻辑）

```typescript
// main.ts _createBattleScene() 伪代码

// 1. 确定地图和章节
if (mode === JOURNEY) {
  boardTemplate = getBoardTemplateForLevel(level.levelId);
  chapterId = level.chapter;
} else {
  const rolled = rollDefenseChapter();     // 守护模式随机地图
  chapterId = rolled.chapterId;
  boardTemplate = getBoardTemplateForChapter(chapterId);
}

// 2. 狮驼岭特殊：锁格+30%
if (chapterConfig.specialMechanic.type === 'stats_boost') {
  lockedCells.push(...额外锁格);
}

// 3. 棋盘初始化（传入章节 ID，启用主题配色）
this.gridMgr.init(boardTemplate, chapterId);

// 4. 机制系统初始化
if (mode === JOURNEY && chapterConfig.specialMechanic.type !== 'none') {
  this.chapterMechanicSystem = new ChapterMechanicSystem(mechanic, battle, grid);
} else if (mode === DEFENSE) {
  this.compositeMechanicManager = new CompositeMechanicManager(battle, grid);
  // 波次 11/21/31 → 依次叠加随机机制
}

// 5. transformWave 中应用狮驼岭数值加成
if (stats_boost) {
  hpMultiplier *= 1.15; attackMultiplier *= 1.15;
}

// 6. 每帧 update 中 tick 机制系统
this.chapterMechanicSystem?.update(dt);
this.compositeMechanicManager?.update(dt);
```

### 5.6 守护模式波次递增规则

```
波次 1-10  → 无机制（热身阶段）
波次 11-20 → 从 DEFENSE_MECHANIC_POOL 随机抽 1 个机制激活
波次 21-30 → 再叠加第 2 个不重复的随机机制
波次 31+    → 再叠加第 3 个不重复的随机机制（地狱模式）
             最多 3 个机制并行，超过不再叠加
```
