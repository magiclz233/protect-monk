## 一、设计理念

### 1.1 目标

所有关卡共用一套棋盘配色和锁定格样式，缺乏章节辨识度。改为每章一章一景，棋盘色彩、障碍物造型、特殊机制全部拟合《西游记》原著对该地点的环境描写。

### 1.2 三合一设计

每章设计三个维度联动：

| 维度 | 说明 | 影响范围 |
|------|------|---------|
| **棋盘配色** | 面板底色、可布阵格、路径格、锁定格的颜色 | GridManager._drawBackground() |
| **障碍物** | 锁定格的造型，支持 AI 生成 80×80 PNG | GridManager._drawLockedCell() |
| **特殊机制** | 部分关卡的额外规则（持续伤害、减速、复活等） | ChapterMechanicSystem |

### 1.3 棋盘配色自动推导规则

```
面板填充 = 玄青 #1a1814（统一不变）
面板边框 = 章节主题色，降低 alpha 至 42%
可布阵格 = 战场底色，alpha 90%
可布阵边框 = 路径色 提亮 15%，alpha 60%
路径格 = 路径色 加深 20%，alpha 90%
路径格边框 = 朱砂 #c43d30，alpha 50%（统一）
锁定格 = 路径色 加深 30% + 去饱和 20%，alpha 98%
锁定格边框 = 章节主题色，alpha 65%
路径宽线 = 绛红 #8b1a1a 暖化，alpha 84%
路径中线 = 章节暗部色 暖化，alpha 76%
路径金线 = 鎏金 #f0c15a，alpha 88%（统一不变）
```

## 五、守护模式协同设计

### 5.1 设计理念

守护模式不同于八十一难的固定关卡推进，定位为**无限挑战 + Roguelite 变数**：

> **每局守护模式 = 随机一张九章地图 + 按波次递增叠加随机机制 + 无尽波次**

### 5.2 地图随机

```typescript
// src/config/ChapterConfig.ts
export function rollDefenseChapter(): { chapterId: number; name: string } {
  const chapterId = Phaser.Math.Between(1, 9);
  const config = getChapterConfig(chapterId);
  return { chapterId, name: config?.name ?? '五行山' };
}
```

每局守护模式开始时，从九章中随机选一张地图：
- **棋盘配色**：随地图章节自动切换
- **锁定格障碍物**：随地图章节自动切换
- **路径布局**：使用对应章节的 BoardTemplate

### 5.3 机制递增规则

```
波次 1-10  → 无机制（热身阶段）
波次 11-20 → 激活第 1 个随机机制（从 7 种机制池中抽取）
波次 21-30 → 叠加第 2 个不重复的随机机制
波次 31+    → 叠加第 3 个不重复的随机机制（地狱模式）
```

机制池（排除 `none` 和 `stats_boost`）：

| 机制 type | 效果 | 参数 |
|-----------|------|------|
| `silence` | 流沙陷足 | 每 15s 沉默 3s |
| `respawn` | 白骨复生 | 30% 概率复活 |
| `fire_zone` | 三昧火灼 | 每 10s 随机 2 格喷火 3s |
| `burn_all` | 烈焰灼烧 | 每秒扣 1%HP |
| `slow` | 激流缓行 | 移速 -20% |
| `root` | 蛛网缠绕 | 每 12s 定身 3s |
| `shield_stack` | 佛光庇护 | 每 20s 敌人 +10% 减伤 |

机制通过 `CompositeMechanicManager` 并行管理，每个机制有独立的 `ChapterMechanicSystem` 实例。

### 5.4 与八十一难的差异

| 维度 | 八十一难 | 守护模式 |
|------|---------|---------|
| 地图 | 固定（关卡→章节） | **随机九章之一** |
| 机制 | 章节固有机制 | **按波次递增叠加随机机制** |
| 难度 | 按关卡线性增长 | 波次递增 + 机制组合 |
| 目标 | 通关 Boss 关 | 存活尽可能多波次 |
| 奖励 | 英雄/法宝/灵蕴 | 排行榜 + 灵蕴 + 碎片 |

### 6.2 GridManager 改造细节

```typescript
// init() 签名扩展
init(templateOrCellSize: BoardTemplate | number, chapterId?: number): void

// 内部流程
// 1. 根据 chapterId 查 ChapterConfig，取 boardColors
// 2. 若未传入 chapterId，回退到 DEFAULT_BOARD_COLORS

// _drawBackground() 配色替换
// 旧：const color = isLocked ? 0x5a4530 : isPath ? 0x4a2020 : 0x6b5540;
// 新：const color = isLocked ? colors.lockedFill : isPath ? colors.pathFill : colors.buildableFill;
//     const borderColor = isLocked ? colors.lockedBorder : isPath ? colors.pathBorder : colors.buildableBorder;

// _drawLockedCell() 三级降级
// 1. 章节专属图片：chapters/ch{N}_locked.png → 加载成功
// 2. 通用图片：五行山-符印封石 → textures.exists()
// 3. 代码绘制：_drawGenericLockIcon() → Graphics 绘制锁图标
```

### 6.3 ChapterMechanicSystem 实现细节

**初始化时**根据 `mechanic.type` 做一次性 setup：
- `respawn`：注册 `ENEMY_KILLED` 事件监听
- `burn_all`：`setGlobalHealMultiplier(0.5)`
- `slow`：`setGlobalSpeedMultiplier(0.8)`
- 其他：无一次性 setup

**每帧 update(dt)**：
- `none`：直接 return
- `silence` / `root` / `shield_stack`：`_updateTimerEffect()` 定时触发
- `fire_zone`：`_updateFireZone()` 管理预警/燃烧/扣血全生命周期
- `burn_all`：`_updateBurnAll()` 每秒 tick 扣血
- `respawn` / `slow` / `stats_boost`：不需要 update

**销毁时**自动清理事件监听 + 重置全局状态。

### 6.4 狮驼岭特殊处理

狮驼岭（Ch8）的 `stats_boost` 机制不通过 `ChapterMechanicSystem` 实现，而是在 `main.ts` 中分两处处理：

```typescript
// 1. 锁格 +30%：在 _createBattleScene() 中
if (chapterConfig?.specialMechanic.type === 'stats_boost') {
  const boostCount = Math.floor(lockedCells.length * 0.3);
  // 从非路径、非已锁、非开放格中随机选额外锁格
  lockedCells.push(...extraCells);
}

// 2. 敌人数值 +15%：在 transformWave 中
const statsBoostMult = 1.15;
hpMultiplier *= statsBoostMult;
attackMultiplier *= statsBoostMult;
```

## 八、扩展方向

1. **Boss 专属棋盘**：每章第 9/18/… 等 Boss 关使用强化版配色（面板边框加辉光等）
2. **守护模式每日挑战**：用日期做 seed，每天固定一张地图，排行榜同日可比
3. **守护模式结算增强**：展示本局地图名 + 激活机制列表 + 存活波次
4. **守护模式专属奖励**：每 5 波随机掉落临时法宝（仅本局有效）
5. **动态天气**：部分关卡随机天气变化（流沙河起雾、火焰山热浪、通天河暴雪）
6. **章节成就**：每章完成后根据机制应对表现给额外奖励（火焰山不带避火罩通关等）
7. **二周目机制升级**：loop 2+ 时章节机制数值提升（间隔缩短、持续时间延长）
